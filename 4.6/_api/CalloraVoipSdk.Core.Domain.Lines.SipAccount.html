AcceptTrunkInbound

Whether the line accepts inbound INVITEs delivered by its registrar/proxy peer or, only when the source is unknown, addressed to its registered domain (SIP-trunk behavior). When true (default) a call for the exact username is always accepted, plus — when no InboundNumbers whitelist is set — calls from the trusted registrar peer and domain-addressed calls from an unknown source. Set to false for a strict 1:1 user account that must accept only its own username. Ignored when InboundNumbers is set (whitelist wins).

InboundNumbers

Optional inbound number (DID) whitelist for SIP trunks. When set, the line only accepts inbound INVITEs whose called number (To user-part on the registered domain) is in this list — useful to disambiguate multiple lines on the same provider domain. When null or empty, the line accepts calls for its exact username, calls delivered by the registrar it registered to, and any number on its registered domain (trunk default). Username-only accounts are unaffected.

public IReadOnlyList<string>? InboundNumbers { get; init; }

Password

SIP account password. Optional: it is only needed when the registrar challenges the registration (401/407). Leave it empty for accounts that do not authenticate — for example an IP-authenticated trunk, or a registrar that does not challenge. If a challenge does arrive and no password is set, registration fails with a clear error.

PublicMediaHost

Optional public IP address to force into the SDP media connection line (c=) and RTP bind for calls on this line. By default the media address is auto-resolved from the OS routing table and NAT is handled by symmetric RTP; set this only behind CGNAT / a static 1:1 NAT with port preservation where the peer does not latch to the source address. Must be an IP literal; non-IP values are ignored. null keeps the auto-resolved media address (default, unchanged behavior).

public string? PublicMediaHost { get; init; }

PublicSipHost

Optional public host (IP or FQDN) to advertise in the REGISTER Contact and Via sent-by instead of the auto-resolved local address. Required behind NAT for public SIP trunks (e.g. sipgate), whose registrar would otherwise bind the number to an unroutable private LAN address and mark the line offline. null keeps the local address (LAN/direct scenarios).
