Interface IPeerConnection

Namespace
CalloraVoipSdk.WebRtc
Assembly
CalloraVoipSdk.Client.dll

A signalling-neutral WebRTC peer connection: the app owns the signalling channel (WebSocket/HTTP/ Callora) and exchanges SDP through it, while this peer runs ICE, DTLS-SRTP, BUNDLE and the RTP/RTCP transport. Encoded media flows through SendAudioAsync(ReadOnlyMemory<byte>, CancellationToken)/SendVideoFrameAsync(ReadOnlyMemory<byte>, uint, CancellationToken) — the SDK is transport-only, so the app owns the codec. The central per-connection abstraction of the WebRTC facade, mirroring the SIP ICall.

GatherCandidatesAsync(CancellationToken)

Gathers server-reflexive ICE candidates (RFC 8445 §5.1.1) from the configured STUN servers, each surfaced on LocalIceCandidateDiscovered to trickle out (RFC 8838). No-op when no STUN servers are configured. Call after producing the offer/answer and before StartAsync(CancellationToken) (the query shares the media socket the transport takes over once started; calling it after StartAsync(CancellationToken) throws). On loopback (the STUN server on the same host) the reflexive address equals the host candidate; redundant-candidate pruning (RFC 8445 §5.4) is a later slice.

SendDtmfAsync(byte, int, CancellationToken)

Sends one out-of-band DTMF tone (RFC 4733 telephone-event) on the peer's audio track. A no-op is not possible: telephone-event must have been negotiated, otherwise this throws. The tone is streamed as an event burst on the audio stream's RTP clock, suppressed until the DTLS handshake keys the transport.

Task SendDtmfAsync(byte toneCode, int durationMs = 160, CancellationToken cancellationToken = default)

Parameters

toneCode byte

The DTMF event code (0–9, 10=*, 11=#, 12–15=A–D per RFC 4733 §3.2).

durationMs int

The tone duration in milliseconds (default 160; at least the RFC 4733 floor).

cancellationToken CancellationToken

LocalIceCandidateDiscovered

Raised as each local ICE candidate is gathered (RFC 8838 trickle), carrying the RFC 8829 candidate: line so the app can signal it to the peer out-of-band. Pair with AddIceCandidateAsync(string, CancellationToken) on the remote side. The host candidate is surfaced at offer/answer time; server-reflexive candidates follow from GatherCandidatesAsync(CancellationToken) when STUN servers are configured.
