TLS configuration for SIP transport connections.

Supports both outbound (client) and inbound (server) TLS use-cases. Certificate loading is lazy and cached after the first call to GetCertificate().

RFC 5922 compliance: set ExpectedSipDomain to enable domain certificate validation per RFC 5922 §7.1 in addition to the standard chain and hostname checks performed by the TLS stack.

ExpectedSipDomain

Optional SIP domain expected in the server certificate's Subject Alternative Name (SAN) extension per RFC 5922 §7.1.

When set, ValidatePeerCertificateSipDomain(X509Certificate2) checks that the peer's certificate contains a dNSName or uniformResourceIdentifier (sip:/sips:) SAN that matches this domain. Leave null to skip RFC 5922 SAN validation.
