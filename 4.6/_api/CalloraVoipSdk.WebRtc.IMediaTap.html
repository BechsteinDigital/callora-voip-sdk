An L3 observer of the encoded media flowing through an IPeerConnection in both directions — the extension seam for recording, analytics, or feeding an AI pipeline without owning the peer. Attach with AttachMediaTap(IMediaTap) and dispose the handle to detach.

public interface IMediaTap

Remarks

Transport-only: payloads are the raw depacketised codec bitstream. A tap is invoked on the media path, so it must return quickly and should not throw — a throwing tap is caught, logged, and isolated from the media flow rather than being allowed to break it.

OnVideo(MediaDirection, ReadOnlyMemory<byte>, uint?, bool, string?)

Observes one video frame in the given direction. rtpTimestamp is null when unknown; isKeyFrame is false for outbound frames (the send path does not carry the flag). rid is the simulcast layer id (RFC 8853) for an outbound simulcast frame, so a tap can tell the layers apart; it is null for a single-stream send and for inbound frames (receive-side RID demux is a later slice). Richer per-frame metadata (MID/SSRC/track ids) will follow when those are surfaced.
