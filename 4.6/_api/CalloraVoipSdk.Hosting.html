Namespace CalloraVoipSdk.Hosting

Classes

StunServerHost

The public STUN server host (Level 2 facade). Assembles the internal CalloraVoipSdk.Core.Infrastructure.Stun.Server.StunServer from a StunServerHostConfiguration — binding the socket on construction — and answers Binding requests with a CalloraVoipSdk.Core.Infrastructure.Stun.Server.StunBindingRequestHandler once started. Kept in the Client layer so the server infrastructure stays an internal implementation detail reached only through this facade.

StunServerHostConfiguration

Immutable configuration a StunServerHost is built from (Level 2). A STUN Binding server needs no credentials; it just returns the client's server-reflexive (mapped) address.

StunServerHostOptions

Mutable options bound from the DI container (Level 3), mapped onto the immutable StunServerHostConfiguration when the host is resolved.

TurnServerCredential

One long-term credential the hosted TURN server accepts (RFC 8656 / RFC 5389 §10.2). The realm is set once on the host configuration (Realm); each credential carries only the username and password.

TurnServerHost

The public TURN/STUN server host (Level 2 facade). Assembles the internal CalloraVoipSdk.Core.Infrastructure.Turn.Server.TurnServer from a TurnServerHostConfiguration — binding the socket on construction — and exposes only the small hostable surface (ITurnServerHost). Kept in the Client layer so the server infrastructure stays an internal implementation detail, reached only through this facade.

TurnServerHostConfiguration

Immutable configuration a TurnServerHost is built from (Level 2). Mirrors the client facades' configuration objects (VoipConfiguration, WebRtcConfiguration): the mutable TurnServerHostOptions maps onto this for the DI happy path.

TurnServerHostOptions

Mutable options bound from the DI container (Level 3), mapped onto the immutable TurnServerHostConfiguration when the host is resolved. Mirrors WebRtcOptions.

Interfaces

IStunServerHost

A hostable STUN server (RFC 5389) for server-reflexive address discovery — the pure-STUN counterpart to ITurnServerHost. It binds its socket on construction (so LocalEndPoint is known immediately, including after an ephemeral bind), answers Binding requests from Start(), and releases everything on disposal.

ITurnServerHost

A hostable TURN relay server (RFC 8656) — the server-side counterpart to the client facades (VoipClient, WebRtcClient). It binds its socket on construction (so LocalEndPoint is known immediately, including after an ephemeral bind), starts serving on Start(), and stops and releases everything on disposal. It serves the TURN methods (Allocate / Refresh / CreatePermission / ChannelBind / Send / Data, plus the TCP extension); it does not answer bare STUN Binding requests — for server-reflexive discovery run an IStunServerHost alongside it.
