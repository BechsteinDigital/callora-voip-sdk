VideoReceiver

Surfaces a call's inbound encoded video frames to the application via the FrameReceived event. Attach to a call to start receiving; a faulting subscriber is isolated so it neither stops other subscribers nor the RTP callback. Mirrors MediaReceiver for video — transport-only, so the payload is encoded, not decoded.

VideoSender

Feeds application-supplied encoded video frames into a call's outbound RTP stream. Attach to a call, then push frames with SendAsync(VideoFrame, CancellationToken). Frames sent while the call is not connected/on-hold are silently dropped rather than throwing. Mirrors MediaSender for video — transport-only, so the payload must already be encoded.

Structs

MediaFrame

One encoded audio frame on the public media contract. The payload carries codec data as negotiated for the call (identified by PayloadType, e.g. 0 = PCMU, 8 = PCMA, 9 = G.722); it is not decoded PCM.

VideoFrame

One encoded video frame on the public media contract. The SDK is transport-only: the payload carries already-encoded codec bytes (one coded picture) as negotiated for the call — it is not decoded pixels, and the SDK never encodes or decodes. Bring your own codec (VP8, H.264, …) and read PayloadType to identify it.

IMediaReceiver

Public per-call tap for inbound media frames. Multiple receivers can be attached to the same call in parallel (e.g. default audio, recording and a realtime bridge); every attached receiver observes every inbound frame.

IMediaSender

Public per-call injection point for outbound media frames. Frames are handed to the call's active send path; the payload must already be encoded in the negotiated codec (see MediaParameters for payload type and clock rate).

IVideoReceiver

Public per-call tap for inbound encoded video frames. Multiple receivers can attach to the same call in parallel; every attached receiver observes every reassembled inbound frame. The SDK is transport-only — decode the payload with your own codec.

IVideoSender

Public per-call injection point for outbound encoded video frames. Frames are handed to the call's active video send path; the payload must already be encoded in the negotiated codec (see MediaParameters for the negotiated media). The SDK is transport-only and never encodes.
