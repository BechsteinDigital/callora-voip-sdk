Class VideoSender

Namespace
CalloraVoipSdk.Core.Application.Media

Feeds application-supplied encoded video frames into a call's outbound RTP stream. Attach to a call, then push frames with SendAsync(VideoFrame, CancellationToken). Frames sent while the call is not connected/on-hold are silently dropped rather than throwing. Mirrors MediaSender for video — transport-only, so the payload must already be encoded.

Events

KeyFrameRequested

Raised when the remote peer requested a keyframe (RTCP PLI/FIR) for the attached call — force your encoder to emit an intra frame next and send it via SendAsync(VideoFrame, CancellationToken) with IsKeyFrame set. Stops firing on Detach()/dispose. Typical usage: sender.KeyFrameRequested += (_, _) => encoder.ForceKeyFrame();

RecommendedBitrateChanged

Raised when the SDK's recommendation for the attached call changes — subscribe and adjust your encoder's target bitrate. Stops firing on Detach()/dispose. The typical usage is the whole point of the API: sender.RecommendedBitrateChanged += (_, e) => encoder.SetBitrate(e.RecommendedBitrateBps);
