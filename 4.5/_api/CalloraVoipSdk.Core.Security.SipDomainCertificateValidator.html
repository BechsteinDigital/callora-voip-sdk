Class SipDomainCertificateValidator

Validates X.509 certificates against SIP domain identities per RFC 5922 "Domain Certificates in SIP".

RFC 5922 §7.1 requires that when a SIP entity establishes a TLS connection, it MUST verify the server certificate contains a subjectAltName (SAN) extension with a value that matches the expected SIP domain. Two SAN entry types are valid:

  1. uniformResourceIdentifier — a sip: or sips: URI whose host component matches the SIP domain (case-insensitive DNS comparison).
  2. dNSName — a DNS hostname that matches the SIP domain, with wildcard support for the leftmost label (e.g. *.example.com).

This validator is intentionally stateless and purely functional so that it can be used from callback contexts (e.g. SslStream certificate validation callbacks) without thread-safety concerns.

Validates that the provided certificate is appropriate for the given SIP domain per RFC 5922 §7.1.

public static bool ValidateSipDomain(X509Certificate2 certificate, string sipDomain)

Parameters

certificate X509Certificate2

The X.509 certificate to validate. Must not be null.

sipDomain string

The SIP domain to match against (e.g. example.com, sip.example.com). Must not be null or whitespace.

Returns

bool

true if the certificate contains a SAN entry that matches sipDomain; false otherwise.

Remarks

Per RFC 5922 §7.1: "A SIP implementation MUST check the subjectAltName extension first; if the extension is present and contains the appropriate SIP domain identity, the check succeeds."
